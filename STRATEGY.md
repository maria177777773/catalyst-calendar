## 1. Большая цель

### Что делаем
Строим **Catalyst.AI** — отдельное AI-направление агентства Catalyst на поддомене `ai.catalystrussia.ru`. Три продуктовые линии:
- **AI-тимбилдинг** (корпоративы с нейросетями)
- **AI-обучение** (курсы и практикумы)
- **AI-развлечения** (вечеринки, квесты, инсталляции)

### Стратегия
Не просто лендинг — **SEO-фабрика на 500+ страниц**, покрывающая все релевантные поисковые запросы в русскоязычной AI-event нише. Короткий путь в ТОП Яндекса и Google через массовое производство качественных посадочных.

### Почему это сработает
- Ниша молодая, конкуренция слабая (#19 — реверс-инжиниринг топ-10 покажет точные цифры).
- У Catalyst есть реальная экспертиза и кейсы — контент не пустой.
- AI-тренд в event-индустрии только набирает — быть первым в SEO = захватить категорию.
- Автоматизация через Claude + скиллы = скорость производства, недоступная конкурентам.

## 3. DNS-квест

Это был самый сложный и поучительный эпизод. Сохраняем полный разбор.

### Проблема
Нужно направить `ai.catalystrussia.ru` на Vercel (`76.76.21.21`), но:
- DNS родительской зоны `catalystrussia.ru` делегирован на Tilda.
- Tilda в UI позволяет добавлять ТОЛЬКО TXT и MX записи.
- Нужна A-запись для поддомена — Tilda её через UI не даёт.

### Что пробовали (хронология)
1. **Добавить CNAME в Tilda UI** → в выпадашке нет CNAME. Провал.
2. **Связать Tilda с Vercel через "Вторичный домен"** → Tilda требует весь домен, не поддомен. Провал.
3. **Определить регистратора через whois / reg.ru / nic.ru** → везде капча. Провал.
4. **Зайти в ЛК хостингов Марии** → нашли Timeweb как регистратора. ✅
5. **Создать поддомен в Timeweb и настроить A-запись там** → изначально думали что не сработает (потому что NS на Tilda). Мария настояла — **СРАБОТАЛО**. Timeweb делает child-zone delegation и берёт на себя DNS поддомена ai, несмотря на то что родительская зона на Tilda.

### Ключевой инсайт
**Child-zone delegation** — в родительской зоне (Tilda) нужны NS-записи для поддомена, указывающие на Timeweb-серверы. Тогда запрос на `ai.catalystrussia.ru` идёт: корневые DNS → Tilda (родительская зона) → видит NS → перенаправляет на Timeweb → Timeweb отдаёт A-запись 76.76.21.21.

**Сейчас:** Timeweb хранит правильную A-запись, НО Tilda не знает о делегировании. Поэтому DNS не резолвится. Нужно попросить Tilda-саппорт добавить:
```
ai.catalystrussia.ru. NS ns1.timeweb.ru
ai.catalystrussia.ru. NS ns2.timeweb.ru
```

### Обходные пути, если Tilda откажет
1. **Перенести ВСЕ NS на Timeweb** — сменить у регистратора NS с Tilda на Timeweb. Но тогда придётся в Timeweb перенастроить ВСЕ записи основного catalystrussia.ru (A, MX, TXT) иначе основной сайт Tilda отвалится. Делается один раз, но с риском.
2. **Cloudflare как промежуточный DNS** — добавить домен в Cloudflare, сменить NS у регистратора на Cloudflare. Cloudflare даёт полный контроль над DNS + бесплатный CDN + SSL. Это правильный долгосрочный путь, но требует переноса всех записей.
3. **Отдельный домен для AI** — зарегистрировать `catalyst-ai.ru` или `catalyst.ai` и не мучиться с делегированием.

### Готовый текст для Tilda-саппорта
```
Здравствуйте! Домен catalystrussia.ru управляется через Tilda DNS (ns1.tildadns.com, ns2.tildadns.com). 
Мне нужно делегировать поддомен ai.catalystrussia.ru на сторонний DNS (Timeweb), 
сохранив управление основным доменом в Tilda.

Прошу добавить в DNS-зону catalystrussia.ru две NS-записи:
ai.catalystrussia.ru. IN NS ns1.timeweb.ru
ai.catalystrussia.ru. IN NS ns2.timeweb.ru

Это child-zone delegation — поддомен ai будет обслуживаться DNS-серверами Timeweb, 
а основной домен останется в Tilda. Записи для основного сайта не меняются.

Спасибо!
```

### Диагностические команды
```bash
# Что видят публичные DNS
dig @8.8.8.8 ai.catalystrussia.ru +short
dig @1.1.1.1 ai.catalystrussia.ru +short

# Что хранит Timeweb напрямую
dig @ns1.timeweb.ru ai.catalystrussia.ru +short     # должно вернуть 76.76.21.21
dig @ns2.timeweb.ru ai.catalystrussia.ru +short

# Что хранит Tilda напрямую
dig @ns1.tildadns.com ai.catalystrussia.ru ANY +noall +answer

# Кто авторитативен для родительской зоны
dig catalystrussia.ru NS +short

# Полная трассировка
dig ai.catalystrussia.ru +trace

# Проверка HTTPS после пропагации
curl -I https://ai.catalystrussia.ru
```

### Факты про DNS (чтобы не путаться)
- **A → 76.76.21.21** — официальный IP Vercel для анонимных доменов.
- **CNAME → cname.vercel-dns.com** — альтернатива A-записи, но для корневого домена работает только A.
- **TTL 600 = 10 минут** — минимальное время пропагации. Реально публичные DNS могут кешировать до часа.
- **Vercel SSL** — автоматический (Let's Encrypt), провижинится 10-60 мин после того как Vercel увидит корректный DNS.

## 4. SEO-фабрика

### Концепция
Не делать 10 страниц руками — строить **фабрику** из шаблонов + данных, которая производит 500+ качественных посадочных под целевые запросы.

### Архитектура
```
[Семантика из MOAB + Вордстат + Rush]
       ↓
[Матрица 500 страниц — Excel]
       ↓
[HTML-шаблон в webflow-стилистике]
       ↓
[Скилл catalyst-seo-factory — генератор]
       ↓
[500 HTML-файлов → Vercel deploy]
       ↓
[Submit в GSC + Яндекс.Вебмастер]
```

### Семантика — источники
- **MOAB.Tools** — зарегистрирован, первичная семантика собрана (задача #33).
- **Яндекс.Вордстат** — автономный Python-парсер работает (#35), 47 маркеров (#37 запустить Rush Analytics).
- **Ahrefs** — первый crawl catalystrussia.ru запущен (#22), Health Score получен. Удалить дубль www (#23).
- **Keys.so** — оценить под задачу (#36).

### Шаблон посадочной (готов)
- Hero + УТП
- Про нас (блок)
- 3 превью направлений
- Bento-блок (программы)
- FAQ
- CTA
- Встроен bubble-меню

Существующие HTML готовы: `teambuilding.html`, `obuchenie.html`, `razvlecheniya.html`, главная переделана (#47, #48, #49, #50).

### Скилл catalyst-seo-factory (#30, в работе)
**Вход:** Excel-матрица со столбцами `URL, H1, Title, Description, Intent, Keyword, Section, ContentBlocks`.
**Процесс:** Jinja2-шаблон + данные → HTML-файл.
**Выход:** 500 HTML-файлов в структуре `/teambuilding/*.html`, `/obuchenie/*.html`, `/razvlecheniya/*.html`.
**Валидация:** проверка уникальности H1/Title/Description, минимального объёма контента, наличия внутренней перелинковки.

### Аналитика — до запуска
Задача #13 — ДО выкладки 500 страниц подключить:
- Яндекс.Метрика (счётчик в `<head>`)
- Google Search Console (верификация через DNS TXT или HTML-файл)
- Яндекс.Вебмастер (верификация)

Иначе первые недели данных потеряем. Есть план сделать скилл `analytics-inject` (#41) — автоматически вставляет все счётчики во все страницы.

## 7. Лайфхаки

Собраны в процессе работы — не забывать использовать!

### Деплой и автоматизация
- **GitHub через REST API вместо git push** — быстрее, не нужен git config, работает из Cowork из коробки. `curl PUT /contents/file` с base64.
- **Vercel deploy через Python** — скрипт `deploy.py` + токен в `.catalyst-deploy.conf`. Одна команда = деплой.
- **HTML→PDF через Playwright** — Chromium headless даёт идеальный PDF без искажений. `wkhtmltopdf` глючит на современном CSS.

### DNS и домены
- **Регистратор через капчу не определишь** — быстрее зайти в личные кабинеты хостингов.
- **Child-zone delegation** — можно управлять DNS поддомена отдельно от основной зоны через NS-записи.
- **TTL 600** при добавлении новых записей — 10 минут пропагации вместо часов.
- **dig @NS напрямую** — обходит кеш публичных DNS, видит реальное состояние.

### Работа с Tilda
- **Ограничения UI:** только TXT/MX в DNS. Всё остальное — через саппорт.
- **Саппорт отвечает быстрее** если сразу дать техническую формулировку (имена серверов, тип записей), а не описание проблемы словами.

### SEO
- **Матрица страниц в Excel** — не писать 500 страниц по одной, строить фабрику.
- **Уникальность контента** — не копипаст с вариациями. Каждая страница — свой угол, свой интент.
- **Внутренняя перелинковка** — между направлениями (teambuilding → obuchenie → razvlecheniya) добавляет вес всем разделам.
- **Аналитика ДО запуска** — чтобы собрать поведенческие с первого дня.

### Claude и Cowork
- **Скиллы > промты** — закрепляет процесс, работает стабильно между чатами.
- **Конспект при переезде** — автоматический через chat-handoff.
- **maria-rules** — обязательный к применению скилл для каждого ответа.
- **GitHub как хранилище документации** — не только код, но и большие стратегические документы, которые переживут любой чат.

### Обновить файл в GitHub
```bash
TOKEN="ghp_XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX"
REPO="maria177777773/catalyst-calendar"
FILE="calendar.js"

# Получить SHA
SHA=$(curl -s -H "Authorization: token $TOKEN" \
  https://api.github.com/repos/$REPO/contents/$FILE | jq -r .sha)

# Закодировать новый контент
CONTENT=$(base64 -w0 новый_файл.js)

# Обновить
curl -X PUT -H "Authorization: token $TOKEN" \
  -H "Content-Type: application/json" \
  https://api.github.com/repos/$REPO/contents/$FILE \
  -d "{\"message\":\"update\",\"content\":\"$CONTENT\",\"sha\":\"$SHA\"}"
```

### Создать новый файл в GitHub (без SHA)
```bash
TOKEN="ghp_XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX"
curl -X PUT -H "Authorization: token $TOKEN" \
  -H "Content-Type: application/json" \
  https://api.github.com/repos/maria177777773/catalyst-calendar/contents/STRATEGY.md \
  -d "{\"message\":\"add strategy doc\",\"content\":\"$(base64 -w0 strategy.md)\"}"
```
